The values of this enumeration specify the different ways of computing the radius of the combined surfel when merging two input surfels.

Enumerator

Compute the radius of the combined surfel as a confidence-weighted average of the radii of the two input surfels.

RCM_DISTANCE 

Compute the radius of the combined surfel as the distance between the positions of the two input surfels.

Updates the depth buffer for an index image by projecting a surfel into the coordinate system of the index image and writing its depth value into the depth buffer if it is closer than the current nearest surfel along its ray.

Note that this does not change the index image itself, it merely fills in the depth buffer so that the index image can then be efficiently updated in parallel.

Parameters
surfelIdThe ID of the surfel being projected.
surfelsThe surfels in the scene.
invTA transformation mapping global coordinates to live 3D depth coordinates.
intrinsicsThe intrinsic parameters of the depth camera.
indexImageWidthThe width of the index image.
indexImageHeightThe height of the index image.
scaleFactorThe scale factor by which the index image is supersampled with respect to the depth image.
useRadiiWhether or not to render each surfel as a circle rather than a point.
unstableSurfelRenderingModeWhether to always/never render unstable surfels, or render them only if there's no stable alternative.
stableSurfelConfidenceThe confidence value a surfel must have in order for it to be considered "stable".
unstableSurfelZOffsetThe z offset to apply to unstable surfels when trying to ensure that they are only rendered if there is no stable alternative.
depthBufferThe depth buffer for the index image.
